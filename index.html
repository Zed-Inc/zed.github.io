Project idea

Overview

Many times people have gone into their local book shop and bought a book basing the story off of the cover art, description or review located on the backside of the book. But these descriptions and reviews are not always right sometimes these are made to make the book sound better than reality. This app aims to help in those decisions by using your phone’s camera to scan a book cover where it will then pull up a selection of reviews from people who have really read the book so you can make an informed decision.


Motivation

Motivation for this project came about after having spent hundreds of dollars on books that
have glowing reviews and descriptions that draw you in located on the back only to read the book
and find a mess of writing and storylines inside. Unable to find anything that was suitable
or did what I want wanted it to with a simple UI was also a large motivating factor in deciding
to purse this application. This application aims to solve these problems by providing an easy to use
and intuitive interface for helping shoppers find books that they Willa actually enjoy reading.


Description

The main feature of the app is its ability to scan a book and extract the title and author from the photo taken. Once the user scans a book cover the app will then run the image through a coreML text extraction model integrated within the app, this machine learning model has been trained to extract the title and author from hundreds of reference photos. If the model can't extract the title and author(this can be for a variety of reasons such as poor lighting, title and author text placement are in odd spots, etc..) then the user will be prompted to enter those details manually. The goodreads api is used to fetch the book reviews where they are then displayed in a cascading list down the screen which you can then sort by rating/star as you would on the website. The goodreads api was chosen because of the 1,000,000+ books and reviews already on the site and because of its high use and trustworthiness among the literature community.

A smaller but still very useful feature of the app is it's ability to leave a review, so once you have finished reading the book and you want to tell everyone about how good it is you have two options, option number one is to go onto goodreads and search up the book you have and leave a review that way, that takes to long. Option number two is to scan the book and flip over to the review section, here you can write out your review to your hearts content before pressing the 'upload' button, the scanning way is easier and for people with a lot of books definitely makes the review process a lot faster.

A similar feature to the book scanning aspect of the app is its ability to also scan dvd's, in this way we can expand upon the already existing text extraction machine learning model used in the app. To search for movies a simple toggle switch or button will be pressed, on the backend side all this button does is switch out the api used, on the frontend there is no noticeable change. IMDB and rotten tomatoes api’s will be used for fetching reviews because of their high catalogue of movies and reviews

Touching briefly on the user interface of the app, it is designed to be as intuitive, fast and simple to use, the main home screen just features a background which is your camera's view and a simple scan button which you press to take a photo of the book and a book/dvd button toggle. After pressing the scan button the view shifts to a cascading card view of reviews which can be sorted by their stars

Eventually in the future the whole backed will be extracted and turned into a seperate rest service for use across multiple different domains, so using the api you can integrate it into a hardware store app and scan the tags to bring up item information from the store website. These sorts of use cases can be extremely useful in hospitals for scanning containers and products. What started out as a simple book scanning app has now grown into a backend service with a large variety of use cases


Tools and technologies

The technologies used in this app include machine learning, networking and simple app development. The initial version of this app will be designed for the apple iPhone using Swift as the core language of choice, for the user interface design and programming SwiftUI will be used because of its simplicity and hot reload capabilities in the Xcode editor. A simple machine learning model will be made with CoreML apples machine learning framework. Networking will be done by using either the inbuilt networking system in swifts Foundation library or using Alamofire. All these technologies have pages of documentation and examples for developers.


Skills required

The skills needed to develop this app are basic programming knowledge with Swift and some knowledge about networking and making api calls. Apple provides a dashboard like interface for CoreML so you don't have to do any programming to setup your model. On the networking side of things knowledge on how to setup an api call, decoding the response into an appropriate data structure and handling errors is a must. SwiftUI is an intuitive user interface framework that once you get over that initial hurdle is simple to use and makes it easy to design complex screens and interfaces.
